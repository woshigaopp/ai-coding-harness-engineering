# 用户写作风格（仅用于写 AIP 文档时）

写 AIP 文档时，按照以下语言习惯：

1. 从目的出发，倒推设计：先说"诞生的目的是解决什么问题"，然后从终点倒推每一步需要做什么
2. 逻辑链条不断：用"因此"、"接下来思考"、"那么"等衔接词，前一步的结论是后一步的输入，不跳步
3. 用具体例子辅助抽象概念：讲到抽象概念时立刻举具体例子（如"以 MySQL 为例"）
4. 先具体再抽象总结：先走完具体推导过程，最后用"总结来看"收束成精炼结论
5. 编号列举关键要素：用 1、2、3 明确列出条件或步骤
6. 用"如果...则..."的条件句处理分支逻辑
7. 中文为主，技术术语保留英文（如 Producer、ProducerRecord、Connect）
8. 语气平实偏书面，不用感叹号，不用口语化表达
9. 段落之间紧凑，不用大量空行分隔

注意：此风格仅在写 AIP 文档时使用，其他场景不适用。
